Отлично, что вы успешно запустили контейнеры! Это действительно огромный шаг к деплою. Использование Docker-образов для размещения на сервере - это современный и очень удобный подход.
Теперь давайте разберем шаги для размещения вашего приложения на платформе Selectel с использованием Docker-образов, а также поговорим о стоимости.
Какие шаги необходимо предпринять для размещения на Selectel?
Предположим, вы используете стандартный подход: арендуете виртуальный сервер (виртуальную машину) в Selectel и запускаете на нем свои Docker-контейнеры с помощью Docker Compose.
Вот основные шаги:

Подготовьте Docker-образы:

Убедитесь, что ваш Dockerfile для приложения и docker-compose.yaml готовы для продакшн-среды (например, не используют отладочные флаги, указывают рабочую базу данных).
Важно: Вам нужно место для хранения ваших образов, чтобы виртуальный сервер Selectel мог их скачать. Самый удобный способ - использовать Docker Registry. Selectel предлагает свой собственный Selectel Container Registry.
Создайте репозиторий в Selectel Container Registry.
Соберите ваш образ приложения локально (docker build -t <имя_образа> . или используйте docker-compose build).
Присвойте образу тег для Selectel Registry: docker tag <локальное_имя_образа> <региональный_адрес_registry>/<имя_репозитория>/<имя_образа>:<тег> (например, cr.ru-1.selectel.cloud/my-app-repo/flask-app:latest).
Авторизуйтесь в Selectel Registry из Docker CLI (docker login cr.ru-1.selectel.cloud и используйте логин/пароль или токен API Selectel).
Загрузите образ в Registry (docker push <полный_тег_образа>).
(Опционально, но рекомендуется): Если вы используете образ базы данных (например, postgres) в своем docker-compose.yaml, убедитесь, что он указан как image: postgres:13 (или другая версия) и будет скачан с Docker Hub или другого публичного Registry. Если вы делали кастомный образ БД, его тоже нужно загрузить.
Арендуйте виртуальный сервер (Облачный сервер / VDS) в Selectel:

Перейдите в панель управления Selectel.
Выберите сервис "Облачные серверы" или "Виртуальные серверы".
Создайте новый сервер.
Выберите конфигурацию: Это самый влияющий на стоимость фактор. Для начала выберите минимально достаточные параметры. Об этом подробнее ниже.
Выберите операционную систему: Рекомендуется выбрать дистрибутив Linux, например, Ubuntu Server последней LTS версии или CentOS/Rocky Linux.
Настройте сеть: Выберите публичный IP-адрес, настройте правила безопасности (Firewall / Security Groups) - по умолчанию лучше все закрыть, разрешив только SSH (порт 22) для администрирования и затем порт вашего приложения (например, 80 или 443, или 5000, если будете обращаться напрямую).
Настройте SSH-ключ: Добавьте ваш публичный SSH-ключ для безопасного доступа к серверу.
Подключитесь к виртуальному серверу и настройте его:

Используя SSH и ваш приватный ключ, подключитесь к публичному IP-адресу созданного сервера.
Обновите пакеты системы (sudo apt update && sudo apt upgrade -y для Ubuntu).
Установите Docker: Следуйте официальной инструкции Docker для вашей ОС (обычно включает добавление репозитория Docker и установку пакетов docker-ce, docker-ce-cli, containerd.io).
Установите Docker Compose: Также следуйте официальной инструкции (обычно это скачивание исполняемого файла и помещение его в /usr/local/bin).
Добавьте вашего пользователя в группу docker, чтобы выполнять команды Docker без sudo (sudo usermod -aG docker <ваш_пользователь>). Может потребоваться перелогиниться.
Разместите файлы проекта на сервере:

Скопируйте ваш файл docker-compose.yaml и, возможно, другие необходимые файлы (если они не включены в образ и не используются в Dockerfile) на сервер, например, с помощью scp или rsync.
Создайте файл .env на сервере (в той же директории, что и docker-compose.yaml) с продакшн-значениями переменных окружения, включая DATABASE_URL, используя имя сервиса базы данных в compose-файле как хост (например, DATABASE_URL=postgresql+psycopg2://postgres:ваш_пароль@db:5432/gig_vospit), а также любые секреты приложения. Важно обеспечить безопасность этого файла.
Настройте доступ к Selectel Container Registry (если используете):

На сервере выполните docker login cr.ru-1.selectel.cloud и используйте учетные данные или токен API, чтобы ваш сервер мог скачивать образы из вашего приватного Registry.
Разверните приложение с помощью Docker Compose:

Перейдите в директорию с файлом docker-compose.yaml на сервере.
Запустите контейнеры: docker-compose up -d. Флаг -d запускает контейнеры в фоновом режиме (detached mode). Docker Compose скачает необходимые образы (ваш образ приложения из Registry, образ PostgreSQL из Docker Hub/Registry), создаст сеть, тома и запустит сервисы.
Настройте веб-сервер (рекомендуется):

Хотя ваше Flask-приложение работает на порту 5000 (или другом, указанном в Dockerfile/команде запуска), напрямую "светить" им в интернет не рекомендуется.
Установите на сервере веб-сервер/обратный прокси, такой как Nginx или Caddy.
Настройте его для проксирования запросов с портов 80 (HTTP) и 443 (HTTPS) на порт 5000 вашего контейнера app внутри сети Docker Compose.
Это позволит вам легко добавить SSL-сертификат (Let's Encrypt), обрабатывать статику, сжимать ответы и т.д.
Настройте DNS (если используете домен):

Если у вас есть доменное имя, создайте A-запись, указывающую на публичный IP-адрес вашего виртуального сервера Selectel.

Какую стоимость придется отдать? Что мне надо?
Стоимость в Selectel (как и у большинства облачных провайдеров) рассчитывается исходя из потребляемых ресурсов. Вам потребуется оплачивать:

Виртуальный сервер (Облачный сервер): Это основной компонент. Стоимость зависит от:

Количества vCPU (процессорных ядер): Определяет вычислительную мощность.
Объема оперативной памяти (RAM): Влияет на то, сколько данных приложение и база данных могут держать в памяти.
Объема и типа диска: SSD быстрее и дороже, HDD медленнее и дешевле. Для БД и приложения рекомендуется SSD.
Операционной системы: На некоторые ОС может быть лицензионная плата (например, Windows), но Linux обычно бесплатный.
Региона дата-центра: Цены могут незначительно отличаться.
Способа оплаты: Оплата обычно почасовая с возможностью резервирования ресурсов на длительный срок со скидкой.
Сетевой трафик: Обычно входящий трафик бесплатный, а исходящий тарифицируется по объему. Стоимость зависит от того, сколько данных ваше приложение отдает пользователям.
(Возможно) Управляемая база данных (Managed Database): Selectel предлагает услугу Managed PostgreSQL. Это готовая база данных как сервис, которой не нужно управлять самому (бэкапы, обновления, масштабирование делает провайдер). Если вы решите использовать эту услугу вместо запуска БД в контейнере, вы будете платить за нее отдельно, по тарифам Selectel на управляемые БД (зависит от размера, RAM, IOPS). Для продакшна это часто более предпочтительный вариант, чем самодельный контейнер с БД.
(Небольшая стоимость) Container Registry: Плата берется за объем хранимых образов. Обычно это очень небольшая сумма для нескольких образов.

Оценка стоимости:
Очень сложно назвать точную сумму без знания ваших потребностей и выбранной конфигурации. Однако для небольшого тестового или стартового приложения с невысокой нагрузкой можно ориентироваться на следующие примеры (цены могут меняться, всегда проверяйте актуальные тарифы на сайте Selectel и используйте их калькулятор!):

Минимальный виртуальный сервер: Конфигурация вроде 1 vCPU, 1-2 GB RAM, 10-20 GB SSD может стоить в районе от 500 до 1500 рублей в месяц.
Небольшая управляемая база данных PostgreSQL: Может добавить еще от 1000 до 3000 рублей в месяц в зависимости от размера и конфигурации.
Трафик: Если трафик небольшой (гигабайты), стоимость трафика будет незначительной.

Таким образом, для запуска вашего приложения и базы данных на виртуальном сервере Selectel минимальные ежемесячные расходы могут начинаться примерно от 500-1500 рублей (если БД в контейнере на том же сервере) или от 1500-4500 рублей (если используете Managed DB).
Где смотреть стоимость и рассчитывать:

Зайдите на сайт Selectel: https://selectel.ru/.
Найдите разделы "Цены" или "Калькулятор".
Особое внимание уделите ценам на "Облачные серверы" (VDS) и "Базы данных как сервис" (Managed Databases), если планируете их использовать.
Используйте калькулятор на сайте Selectel, выберите нужные параметры (количество CPU, RAM, тип и размер диска, регион) - он покажет ориентировочную стоимость в месяц.

Итог:
Вам потребуется арендовать Виртуальный сервер (Облачный сервер) в Selectel. На нем вы установите Docker и Docker Compose, скопируете ваш docker-compose.yaml и .env, скачаете образы (лучше из Selectel Container Registry), и запустите приложение. Стоимость будет зависеть от выбранной конфигурации сервера, объема трафика и использования дополнительных услуг вроде Managed Database. Начните с минимальной конфигурации и масштабируйте по мере роста нагрузки.
Удачи с развертыванием! Вы на правильном пути.
